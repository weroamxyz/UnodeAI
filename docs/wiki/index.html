UnodeAi 0.9.11

Multi-model AI teams inside VS Code.

UnodeAi gives each agent its own role, model, chat, tools, memory, and safety policy. Use one Solo agent for quick tasks, a PM-led crew for larger work, or workflows for repeatable processes.

Start setup View commands Model pricing

Quick Start

  1. Install UnodeAi from the VS Code Marketplace or from a `.vsix` file.
  2. Run UnodeAi: Run Setup Wizard.
  3. Run UnodeAi: Set Provider API Key and store ROAM_API_KEY.
  4. Run UnodeAi: Create or Switch Team.
  5. Open UnodeAi: Open Chat with Agent and send a goal to the Project Manager.

The default Roam gateway (weroam) is https://ai.weroam.xyz/v1 and uses OpenAI-compatible chat completions.

Teams and Roles

The default software crew is Project Manager, System Architect, Senior Developer, and Reviewer. You can also add QA, DevOps, Technical Writer, Security, Data Engineer, Product Manager, Solo, and knowledge-work specialists.

Project ManagerDelegates, tracks todos, runs checks, and coordinates review.
ArchitectDefines contracts, boundaries, and non-overlapping file ownership.
Senior DeveloperImplements focused code changes and tests.
ReviewerInspects work independently and returns PASS or FAIL.
SoloOne generalist agent for fast direct work.
SpecialistsQA, DevOps, Security, Data, Technical Writer, and business roles.

Build your own agent

Run UnodeAi: Build an Agent to open the Agent Builder — a form to create or edit a custom agent (name, a built-in or custom role like CEO, icon, model, system prompt, capability tools, and MCP grants) without touching JSON. Attach up to 5 skill playbooks from the skill library: market-proven procedures (OWASP review, dependency-risk triage, test-coverage gaps, accessibility audits…) that fold into the agent's instructions under a ## Playbooks section, so it arrives knowing how to do the job. The in-app library stays lean; the builder links out to the full skill library for the long tail. Save, and the agent joins your team like any preset.

Chat, Plan, and Act

Every agent has a persistent chat. The Chat panel supports live streaming, Stop, Steer, tool cards, diffs, command output, analysis cards, context usage, todos, approvals, and import/export. Clear deletes a transcript; Archive hides it without deleting (restore later via View Archived Chats). Agents can also search_files (regex → file:line) and delete_file (sandboxed + checkpointed) so they don't resort to scratch scripts or blocked shell tricks.

ModeUse it forTool behavior
PlanAnalysis, estimates, design, review, and risk discussion.Write, command, delegation, and MCP tools are removed for OpenAI-compatible agents.
ActImplementation, edits, checks, and approved tool use.Agent can use tools allowed by role, settings, and approvals.

@roam in the VS Code Chat panel

UnodeAi also appears in the native VS Code Chat panel as @roam — alongside its own sidebar, both at once. Type @roam <goal> and your crew's PM picks it up, delegates, and streams the run back into the chat, with an Open in UnodeAi button for the full team view. It runs on your UnodeAi agents/models (not the chat panel's model), so you keep the multi-agent orchestration and cheap-model cost arbitrage. Needs at least one team; toggle with roam.chatParticipant.enabled.

PM Orchestration

Send complex goals to the Project Manager. The PM breaks the work into a live todo list, asks the Architect for contracts when needed, delegates to teammates, collects results, runs checks, and sends work to an independent Reviewer.

Goal: Add password reset.
Scope: API route, email token model, UI form, tests.
Constraints: Keep the existing auth provider.
Verification: npm test and npm run build.

Best results come from clear scope, explicit constraints, and a configured roam.verifyCommand.

Security

  • Provider keys are stored in VS Code SecretStorage.
  • File access is sandboxed to the workspace or assigned worktree.
  • Command execution is controlled by roam.commandApproval.
  • File writes can require diff approval with roam.writeApproval = ask.
  • MCP servers are default-deny and can require approval before mounting.
  • Plan mode removes high-impact tools for OpenAI-compatible agents.
  • Zero data retention & no telemetry. UnodeAi itself keeps no copy of your code or prompts and has no analytics, tracking, or phone-home. Chat history, config, and keys stay on your machine; your code is sent only to the model provider you configure. UnodeAi works with any OpenAI-compatible endpoint, so you can point it at a self-hosted / in-VPC model for provable, end-to-end zero-retention.

Use roam.commandApproval = all only inside a sandbox or disposable environment.

Verified worktree fan-out

For larger parallel jobs, set roam.concurrencyStrategy = worktree (requires a git repo with a clean tree). Each eligible agent gets its own isolated git worktree under .roam/worktrees/; when its turn finishes, its work is committed and merged into a Roam integration branch (roam/integration).

Only work that passes your checks lands. Before merging, UnodeAi runs your roam.verifyCommand (build/lint/test) inside that agent's worktree. If it passes, the work merges; if it fails, the work is held on the agent's own branch and handed back to fix — nothing unverified reaches integration. Neither Cline nor Kilo gate the team merge on verification.

  • Review with UnodeAi: Crew Worktrees (Review) — each lane shows ✓ verified / ✗ failing / ⚠ unverified, and a lane that passed by editing the tests (instead of fixing the code) is flagged for review.
  • Land integration work onto your branch with UnodeAi: Finalize Worktree Merges to Branch.
  • Gate is controlled by roam.worktree.verifyBeforeMerge (on by default) + roam.worktree.verifyTimeoutSeconds; with no roam.verifyCommand there's nothing to gate on and merges proceed.
  • Control parallelism with roam.worktree.maxParallel; leave roam.worktree.autoMerge off when you want manual review.

Troubleshooting

SymptomFix
No API keyRun UnodeAi: Set Provider API Key.
Command blockedCheck roam.commandApproval and roam.allowedCommands.
PM cannot run checksSet roam.verifyCommand and approve the command.
Model rejects optional paramsClear unsupported fields such as reasoning_effort.
Conflicting editsRe-read and retry, or switch larger jobs to worktree mode.
Stale workspace stateUse UnodeAi: Reset Workspace State only when a full reset is intended.
