## 逐项压测

### 1. V1 Checkpoints / Restore（一键还原）
- 让 agent 改 2-3 个文件(新建 + 覆盖已有)。
- 点 Team 面板的 🕘 **Restore File Checkpoint** 按钮(或命令面板 `UnodeAi: Restore File Checkpoint…`)。
- ✅ 通过:QuickPick 倒序列出刚才的文件改动;选一个 → 弹确认 → **文件被还原到改动前**(新建的文件应被删除);
  对话能继续。
- 边界:对**新建文件**还原 = 删除;对**覆盖**还原 = 回到旧内容。两种都试。

### 2. V2 写文件审批（diff 预览 + 批准/拒绝）
- 设置 `roam.writeApproval` = `ask`,重载窗口。
- 让 agent 写文件。✅ 通过:写入前**弹出 diff 预览**(create/overwrite + 统一 diff),三个按钮:
  **Approve / Approve all (session) / Deny**。
  - 点 **Deny** → 写入被拦,文件没变,模型收到"被用户拒绝、别原样重试"的提示。
  - 点 **Approve** → 文件写入。
  - 点 **Approve all** → 之后本会话不再弹窗,直接写。
- 验完把 `roam.writeApproval` 调回 `none`。

### 3. V3 并行看板（Console）
- 起**多个 agent**(团队或多个 Solo)。打开 **Console** 面板。
- ✅ 通过:实时显示每个 agent 的 状态(🏃/🧘/😴/🤒)、当前任务、ctx%、成本/轮次;状态变化即时刷新;
  出错的 agent 显示红色错误行;空团队显示友好空态。

### 4. C3 实时 Todo 自动收起（刚修的 bug)
- 给一个 **≥4 步**的真任务,要求它用 update_todos 列步骤。
- ✅ 通过:顶部 Plan 清单实时 `☐→▸→☑` 走到满;**全部完成后自动收起成一行 `✓ N/N`(绿色)**,聊天区域恢复
  高度;点那一行能重新展开回看。

### 5. A/B 健壮性（弱模型防呆,你最该验的）
- 故意制造一个会让你(DeepSeek)犯错的多步任务(比如让你写一个结构复杂的文件)。
- ✅ 通过:即使你发了**畸形/缺参数的工具调用**,系统**不再诡异死循环**——要么一上来就给"缺少必填参数 X"的
  明确纠正,要么"同样的失败调用连续几次后被拦下并提示换法",而不是卡住。
- 这是你之前 write_file 死循环的根治,重点确认**不会再卡死**。
